2.  在少样本条件下复现典型过拟合现象：训练损失持续下降，验证损失先降后升，模型泛化失效

4.  采用多方法组合策略优化模型，实现过拟合抑制与泛化能力提升，输出可直接用于论文的实验结果与损失曲线

## 三、核心原理收获（可直接写入论文理论基础）
1.  地下水参数反演属于典型的**少样本、高维回归任务**，数据稀疏性与反问题不适定性极易诱发过拟合
2.  过拟合的典型表现：训练集损失持续降低，验证集损失先降后升，模型对训练样本“死记硬背”，但对真实场地数据泛化能力极差，无法用于工程预测
3.  三种防控方法的核心机理与适用场景：
    - Dropout：训练时随机失活神经元，打破特征共线性，降低模型对特定样本的依赖
    - L2正则化：对权重施加惩罚，约束参数空间复杂度，避免权重过度拟合噪声
    - 早停法：在验证损失上升拐点停止训练，防止模型过度学习训练集噪声
4.  实验验证：三种方法组合使用时，过拟合抑制效果最优，能有效平衡模型拟合能力与泛化能力

## 五、遇到的问题与解决办法（可作为论文“方法优化”部分的补充）
1.  **问题1**：训练循环重复编写，导致损失列表长度与绘图维度不匹配报错
    - 解决：精简合并单一训练循环，损失列表长度与实际训练轮数严格对齐，统一数据格式
2.  **问题2**：Dropout层添加后未在前向传播中调用，正则化未生效
    - 解决：在模型的forward方法中显式调用Dropout层，确保训练模式下随机失活生效
3.  **问题3**：早停逻辑位置错误，判断失效
    - 解决：将早停判断放在每轮验证损失计算完成后，执行最优模型保存与轮数计数，避免训练提前中断
4.  **问题4**：模型在验证集上的泛化性能提升不明显
    - 解决：调整L2正则化系数与Dropout失活率，采用组合策略协同抑制过拟合，优化模型结构
