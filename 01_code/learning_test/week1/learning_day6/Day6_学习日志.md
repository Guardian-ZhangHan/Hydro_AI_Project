## 三、核心原理收获
1.  **数据泄露的防范机制**：归一化器仅用训练集拟合，验证/测试集仅做变换，避免模型提前接触测试数据
2.  **数据一致性校验的必要性**：反归一化误差是衡量数据处理流程无失真的关键指标，误差需控制在1e-6以内
3.  **固定随机种子的重要性**：全局种子固定是确保数据集划分、模型训练结果可复现的核心前提
4.  **数据格式与模型的适配逻辑**：DataLoader输出格式必须与模型输入维度完全匹配，否则会导致训练失败

## 五、遇到问题与解决办法
1.  问题：验证集/测试集归一化值跑出[0,1]区间，断言校验失败
    解决：改用带`clip=True`的MinMaxScaler，并在变换后添加`np.clip`二次截断，彻底解决越界问题
2.  问题：担心本地多次错误运行会同步到GitHub，占用空间
    解决：明确Git提交逻辑：仅手动执行`git add/commit/push`才会上传文件，本地运行不影响GitHub仓库
3.  问题：混淆原始数据与处理后数据的作用，担心数据泄露
    解决：明确数据用途：原始模拟数据仅作备份，训练只用归一化后数据集与训练集拟合的scaler
